# 期末救命箱

面向大学生的课程资料与 DDL 智能整理助手。

「期末救命箱」不是网盘，而是把群聊通知、课程文件、作业要求、复习资料整理成按课程归档的资料库和待办清单。MVP 阶段优先跑通从资料或通知输入，到识别确认、归档、搜索的完整闭环。

## 产品定位

大学生的课程信息常常分散在 QQ 群、微信群、学习平台、邮箱和本地文件夹中，导致资料难找、课程归属混乱、DDL 容易遗漏。

本项目希望把这些碎片信息结构化，形成：

- 按课程组织的资料库
- 可追踪的 DDL 待办清单
- 可确认和修改的通知识别结果
- 面向课程、文件、标签、通知和待办的全局搜索

## MVP 范围

第一阶段必须完成：

- 首页 Dashboard
- 课程列表
- 课程详情页
- 添加资料页
- 粘贴通知页
- AI 识别结果确认页
- 归档后的资料列表和待办列表
- 全局关键词搜索
- 演示用假数据
- localStorage 本地持久化

第一阶段暂不包含：

- 登录注册
- QQ / 微信自动导入
- 学习通接口
- OCR
- 向量数据库
- 文件全文解析
- 多人协作
- 真实云存储
- 日历同步
- App

## 核心用户流程

1. 用户进入首页 Dashboard，查看课程、近期 DDL 和最近资料。
2. 用户创建或选择课程，例如「数据库系统」「高等数学」「大学英语」。
3. 用户添加课程资料，保存文件名、课程、资料类型、标签和上传时间。
4. 用户粘贴群通知文本。
5. 系统使用 MVP mock 识别逻辑提取课程、事项、截止时间、提交方式和标签。
6. 用户进入识别结果确认页，修改或确认识别结果。
7. 系统将确认后的内容归档为通知记录和待办事项。
8. 用户在课程详情页查看该课程的资料、通知和待办。
9. 用户在首页使用全局搜索查找课程名、文件名、标签、通知内容或 DDL 标题。

## 推荐技术栈

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui
- lucide-react
- localStorage
- date-fns 可选

MVP 先以前端可跑通为目标，不引入后端数据库、认证系统、对象存储或向量检索。

## 数据持久化

第一阶段使用 localStorage 保存应用数据。

建议统一保存为一个对象：

```ts
type AppData = {
  courses: Course[]
  materials: Material[]
  todos: Todo[]
  notices: Notice[]
}
```

后续如果接入真实数据库，应保持页面和组件尽量依赖数据访问封装，而不是直接散落调用存储实现。

## 开发原则

- 先完成闭环，再优化局部体验。
- 页面文案使用中文。
- 代码变量名、类型名、文件名使用英文。
- 组件拆分清晰，避免把业务逻辑集中在单个页面文件中。
- AI 识别先使用 mock 规则，后续再替换为真实模型接口。
